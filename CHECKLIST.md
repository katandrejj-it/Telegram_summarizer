## 🐛 Возможные проблемы

### Проблема: Нет сообщений в дайджесте
**Причины:**
1. Чаты неактивны за последние HOURS_BACK часов
2. Все сообщения отфильтрованы (FILTER_MODE = "strict")
3. MIN_MESSAGES слишком высокий

**Решение:**
```python
# В config.py:
MIN_MESSAGES = 1  # Временно снизьте
FILTER_MODE = "off"  # Временно отключите фильтрацию
```

### Проблема: Ошибка "Could not parse JSON"
**Причина:** LLM вернул невалидный JSON

**Решение:** Это нормально, используется fallback. Если часто:
```python
# В summarizer.py увеличьте temperature:
temperature=0.3  # Вместо 0.2
```

### Проблема: Слишком много тем
**Решение:**
```python
# В config.py:
MAX_TOPICS_PER_CHAT = 3  # Вместо 7
MIN_MESSAGES_PER_TOPIC = 5  # Вместо 3
```

### Проблема: Неправильные ссылки
**Причина:** LLM неправильно определил first_message_id

**Решение:** Это улучшится со временем. Пока используется fallback на первое сообщение чата.

### Проблема: Ошибка импорта
```
ModuleNotFoundError: No module named 'tg_digest.message_filter'
```

**Решение:**
```bash
# Проверьте файл:
ls -la tg_digest/message_filter.py

# Если нет — создайте заново из UPGRADE_GUIDE.md
```

## 📊 Мониторинг

### Логи для отслеживания
```bash
# Запустите с подробными логами:
LOG_LEVEL=DEBUG python -m tg_digest.scheduler --once
```

Важные метрики:
- Количество собранных сообщений
- Количество отфильтрованных сообщений
- Количество тем на чат
- Время обработки

### База данных
```bash
# Статистика по сообщениям:
sqlite3 digest.db "
SELECT 
    chat_name, 
    COUNT(*) as msg_count,
    MIN(date) as first_msg,
    MAX(date) as last_msg
FROM messages 
GROUP BY chat_name;
"
```
