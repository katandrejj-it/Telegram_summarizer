## ⚠️ Проблемы и решения:

### Проблема 1: Неправильный ID чата
```
ERROR: Cannot resolve -2189876460: Invalid object ID for a chat
```

**Причина:** ID `-2189876460` имеет неправильный формат.

**✅ ИСПРАВЛЕНО:** Закомментирован в config.py

**Как получить правильный ID:**

#### Вариант 1: Через @userinfobot
1. Перешлите любое сообщение из нужного чата боту @userinfobot
2. Он покажет правильный ID в формате `-100XXXXXXXXXX`

#### Вариант 2: Используйте username
Если у чата есть публичный username (например, @my_channel), используйте его:
```python
CHATS_TO_MONITOR = [
    "my_channel",  # Без @
]
```

#### Вариант 3: Через Telethon
```python
# Временно добавьте в collector.py для отладки:
async with client:
    dialogs = await client.get_dialogs()
    for dialog in dialogs:
        print(f"{dialog.name}: {dialog.id}")
```

### Проблема 2: Groq API Rate Limiting
```
429 Too Many Requests
Retrying in 1.000000 seconds
Retrying in 33.000000 seconds
```

**Причина:** Превышен лимит запросов к Groq API (бесплатный тариф).

**Это нормально:** Groq автоматически делает retry, все запросы успешно выполнены.

**Оптимизация (опционально):**

1. **Добавьте задержку между запросами:**

```python

```

2. **Уменьшите количество обрабатываемых чатов:**

```python
# В config.py:
MIN_MESSAGES = 5  # Вместо 3 — меньше чатов обрабатывается
```

3. **Обрабатывайте чаты по очереди (не все сразу):**
